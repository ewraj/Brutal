# Brutal — Project State File
Read this FIRST before making any changes. Update it after every change.

## Architecture
- **Type**: Client-side web app (HTML/CSS/JS), no build step, no framework
- **AI Backend**: Puter.js (`puter.ai.chat()`) — free Gemini access in-browser
- **Hosting**: GitHub Pages at https://ewraj.github.io/Brutal/
- **Storage**: Puter KV (`puter.kv.get/set`) for chats, settings; localStorage for drafts

## State Object (STATE in app.js)
- `chats` — Object of all chat sessions, keyed by ID
- `currentChatId` — Active chat
- `isThinking` — True during AI generation
- `cancelStream` — Flag to abort streaming
- `currentModel` — e.g. 'gemini-2.0-flash'
- `currentModeKey` — Active persona mode key
- `customPersona` — User's custom persona text
- `pendingAttachment` — File waiting to be sent

## Persona Modes (MODES object)
- `brutal` (default), `codeReview`, `debate`, `essay`, `custom`

## Workflow System (WORKFLOW_PROMPTS object)
- **Workflows**: `teardown` (Tear Apart), `security` (Security Audit), `optimize` (Optimize)
- Each has: `title`, `desc`, `steps[]`, `prompt(code)` function
- Prompts force JSON output with `scores`, `issues[]`, `summary`
- Uses non-streaming `puter.ai.chat()` (streaming had compat issues)
- Results rendered as structured cards: score dashboard + severity-colored issue cards + copy-fix buttons
- Workflow panel visible only in `#chat-wrapper.initial-state`

## Code-First Annotator Mode
- Activated when a workflow completes or user clicks "Annotate" from results
- Split-pane layout: line-numbered code editor (left) + annotations panel (right)
- Annotations are pinned to specific line numbers from the workflow issues
- Clicking a line or annotation cross-highlights both sides
- Responsive: stacks vertically on mobile

## Puter.js API Usage
- `puter.ai.chat(messages, { model, stream })` — Chat/stream
- `puter.ai.chat(messages, { model })` — Non-stream (returns response.message.content)
- `puter.kv.get(key)` / `puter.kv.set(key, value)` — Persistent storage
- `puter.auth.signIn()` / `puter.auth.isSignedIn()` — Auth

## UI Paradigm
- **NOT a chatbot** — looks like a developer tool / IDE console
- Messages render as full-width log entries (no bubbles, no cards, no rounded corners)
- User messages: monospace, dim, left-border line
- Brutal messages: full-width content blocks with no background/border
- Input: terminal prompt bar pinned to bottom edge (monospace, no rounded pill shape)
- Header: compact toolbar with small BRUTAL wordmark (0.85rem, letter-spaced)
- System notice: `// ready` — minimal boot line
- Workflow panel: shows in initial state, disappears once chat starts
- Annotator: full-screen split-pane overlay
